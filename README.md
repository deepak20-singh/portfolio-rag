# Portfolio RAG Chatbot вЂ” Backend

A RAG-powered chat API that answers questions about my work, skills, and experience.
Embedded as a floating widget in my [portfolio](https://myportfolio-dun-psi.vercel.app).

---

## How it works

```

Embed with all-MiniLM-L6-v2
    в”‚
    в–ј
Search FAISS index (top-5 chunks)
    в”‚
    в–ј
Build prompt with retrieved context
    в”‚
    в–ј
Stream response via Groq (Llama-3.3-70B)
    в”‚
```

The knowledge base is compiled from 7 markdown documents covering experience,
projects, skills, education, and FAQ. Embeddings are generated locally with
`sentence-transformers` and stored in a FAISS index.

---

## Stack

| Layer | Technology |
|---|---|
| API | FastAPI + Uvicorn |
| Embeddings | `all-MiniLM-L6-v2` (sentence-transformers) |
| Vector search | FAISS (IndexFlatIP вЂ” cosine similarity) |
| LLM | Groq вЂ” `llama-3.3-70b-versatile` |
| Streaming | Server-Sent Events (SSE) |
| Deployment | Render (free tier) |

## Local setup

**1. Clone and install**
```bash
git clone https://github.com/deepak20-singh/portfolio-rag.git
cd portfolio-rag/backend
pip install -r requirements.txt
```

**2. Set environment variables**
```bash
cp .env.example .env
# Add your GROQ_API_KEY to .env
```

**3. (Optional) Rebuild the knowledge base index**

Only needed if you edit the markdown files in `data/`.
```bash
python ingest.py
```

**4. Start the server**
```bash
uvicorn main:app --reload --port 8000
```

## API

### `GET /health`
Liveness check. Returns model name and total indexed vectors.

```json
{ "status": "ok", "model": "llama-3.3-70b-versatile", "vectors": 33 }
```

### `POST /chat`
RAG-powered chat with SSE streaming.

**Request**
```json
{ "question": "What is your experience with RAG?" }
```

**Response** вЂ” `text/event-stream`
```
data: {"token": "I "}
data: {"token": "have "}
data: {"token": "extensive..."}
data: {"done": true, "sources": ["faq", "experience"]}
```

---

## Deployment

Deployed on Render using `render.yaml`. Set `GROQ_API_KEY` in the Render
dashboard under Environment variables вЂ” everything else is configured in the yaml.

> **Note:** The free tier spins down after 15 minutes of inactivity.
> First request after sleep takes ~30s to wake up.

---

## Updating the knowledge base

1. Edit the relevant file in `backend/data/`
2. Run `python ingest.py` to rebuild the FAISS index
3. Commit the updated `index/` files and push вЂ” Render auto-deploys
